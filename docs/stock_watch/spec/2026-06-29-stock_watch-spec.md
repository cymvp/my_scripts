# A 股自选盯盘工具 — 设计文档（Spec）

- 日期：2026-06-29
- 模块：stock_watch
- 状态：已实现

## 1. 目标

一个常驻屏幕最前端的透明悬浮小窗，实时显示自选股（A 股 + 港股）的名称和涨跌幅，不用切换窗口就能盯盘。支持窗口内增删自选、拖动排序，配置持久化到本地。

## 2. 范围与非目标

**做：**
- 置顶透明悬浮小窗，横向铺开显示自选股的 `名称 涨跌幅%`
- 点 `+` 弹框增、点 `✕` 删、拖名称排序，配置存本地文件
- 支持 A 股（6 位）与港股（5 位）
- 每 3 秒刷新一次行情

**不做（YAGNI）：**
- 不做现价、涨跌额、最高/最低、成交量等其他字段（用户明确只要名称+涨跌幅）
- 不做 K 线图、历史数据、预警提醒
- 不做逐笔（tick 级）实时——免费数据物理上限是 L1 的 3 秒快照，逐笔需付费 Level-2，不在本期范围

## 3. 数据源

**新浪 L1 行情接口**：`https://hq.sinajs.cn/list=<code>`

- 选新浪而非腾讯：新浪直接返回昨收价，涨跌幅算得准；字段顺序比腾讯稳定。
- 请求必须带请求头 `Referer: https://finance.sina.com.cn`，否则被拒。
- 返回 GBK 编码，需转 UTF-8。
- 新鲜度：实测数据时间戳比当前时间晚约 3 秒（= 沪深交易所 L1 快照推送周期，免费数据极限）。

**返回格式**（实测，以 sh600519 为例）：
```
var hq_str_sh600519="贵州茅台,1169.000,1168.630,1195.010,1215.000,1151.010,...,2026-06-29,13:47:26,00,";
```
逗号分隔字段（0-based 索引）：
- `[0]` 名称
- `[1]` 今日开盘价
- `[2]` 昨日收盘价（用于算涨跌幅）
- `[3]` 当前价
- 其余字段本期用不到

**涨跌幅计算**：`(当前价 - 昨收) / 昨收 * 100`，保留 2 位小数。
若昨收为 0 或当前价为 0（停牌/未开盘），涨跌幅显示 `—`。

**批量请求**：新浪支持一次请求多只股票，`list=sh600519,sz000001,...`，逗号拼接，减少请求次数。

## 4. 股票代码识别

按输入数字位数区分市场，自动加前缀：

| 位数 | 市场 | 规则 |
|------|------|------|
| 6 位 | A 股 | 开头 `6/9`→`sh`，`0/2/3`→`sz`，`4/8`→`bj` |
| 5 位 | 港股 | 一律加 `hk` 前缀（需带前导零，如 `02513`、`00700`） |

> 其它位数视为非法输入。

**港股字段位置与 A 股不同**（见下表），`parse_sina_response` 按 `hk` 前缀分支取字段：

| 市场 | 名称 | 昨收 | 现价 |
|------|------|------|------|
| A 股 | `[0]` | `[2]` | `[3]` |
| 港股 | `[1]`（中文名）| `[3]` | `[6]` |

## 6. 数据流

- 启动 → `load_config()` 拿代码列表 → `fetch_quotes()` 首次渲染 → 启动 3 秒定时器
- 定时器每 3 秒 → `fetch_quotes()` → 更新各方块
- 添加：点 `+` 弹框输代码 → `normalize_code()` 校验 → 加入列表 → `save_config()` → 立即刷新
- 删除：点 `−` 弹出勾选列表 → 勾中的批量移除 → `save_config()` → 重渲染
- 排序：拖动股票名称，松手按指针落点重排 → `save_config()` → 重渲染

## 7. 界面

整窗就一行，从左到右：`时间   股票…   +   −`。

- tkinter 窗口：`-topmost` 置顶；`overrideredirect(True)` 去掉原生标题栏；`-transparent` + `systemTransparent` 透明背景（只剩文字浮在桌面）。
- **最左是时间**（`● HH:MM`，绿=已更新 / 红=未更新），兼作：
  - 拖动手柄——按住拖动它移动整个窗口（无标题栏后唯一的移动方式）。
  - 右键弹出"退出"菜单；另绑 Esc 键退出。
- 股票**横向铺开**，每只单行显示 `名称 涨跌幅%`，方块间距小：
  - 名称取**前 2 个字**，蓝色（`#2e9bff`）。
  - 涨跌幅紧跟名称：涨 > 0 红，< 0 绿，= 0/无效 灰。
  - 不再有每股 `✕`（省空间、防误删）。
- 末尾两个无边框按钮（均用 Label 实现，避免 macOS 按钮方框）：
  - `+`：弹输入框添加自选。
  - `−`：弹勾选列表（带标题栏的普通 Toplevel），勾中后"确认删除"批量删除。
- 拖动名称可调整顺序（松手即重排，无拖动中预览）。
- 窗口尺寸随股票数量自适应，整体保持窄小。

## 8. 错误处理

- **网络/接口失败**：保留上一次的涨跌幅，行尾显示一个灰点标记"未更新"，不崩溃；下次定时刷新自动恢复。
- **非法代码**：`normalize_code()` 返回 None → 弹出提示，不加入列表。
- **重复添加**：已在列表中的代码忽略，给个提示。
- **停牌/未开盘**（昨收或现价为 0）：涨跌幅显示 `—`。

## 9. 依赖与运行环境

- **零第三方依赖**：拉接口用 Python 自带的 `urllib`（不用 `requests`），界面用 `tkinter`。
- **解释器**：`/usr/local/bin/python3.10`（Homebrew python@3.10）。
  - 系统自带 `/usr/bin/python3`（3.9.6）虽有 tkinter，但其 Tk 8.5 在 macOS 26 上建窗崩溃（`macOS 26 or later required`），不可用。
  - Homebrew python@3.10 默认不含 tkinter，需一次性安装：`brew install python-tk@3.10`（提供 Tk 8.6）。
- **运行**：`/usr/local/bin/python3.10 stock_watch.py`
